Hotel booking backend

Build a Backend Hotel room booking system

 

Roles

  • Admin

  • User

 

Operations

 

Mandatory

  • Admin can do the following to the rooms in a hotel:

    • Add rooms in a hotel

    • View the rooms in a hotel

    • Remove the rooms in a hotel

    • Update the details of the rooms in a hotel

  • Users can do the following:

    • Book rooms in a hotel

    • View the rooms in a hotel

    • Book random rooms on random days

    • Book list of rooms for particular dates

    • Book a room for list of dates

    • Example

      • Users can book room 1 for 3 days. For example, 31/08/2021, 01/09/2021, 02/09/2021.

      • Users can book room 2 on 31/08/2021, room 3 on 29/08/2021, room 1 on 01/09/2021 

    • Only a single booking API should be used to handle all the use cases

  • A user cannot book a room if it is not available or booked by anyone.

Advanced

  • An API to fetch all available rooms data

  • An API to fetch all booked rooms data

  • If a booking is successful, fetch the total price of rooms booked by the user.

  • Pricing of each room can be changed

  • Admin can fetch the total revenue

Schemas

  • User

  • Room

  • Dates

  • Booking

Unit test cases

Documentation

Host/Deploy

  • Heroku/AWS/Azure etc

Tech Stack 

  • Spring Boot, Java

  • Database - Any database
